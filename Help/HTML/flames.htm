Transforms

 

Transforms are probably the most important elements of fractal flames. Each flame consists of a set of transforms where each transform consists of a set of properties (like weight, color and opacity) and a set of variations.

 

As the name says, transforms perform a transformation. In Apophysis, these transformations are affine - which means that moving, scaling, rotating and shearing is allowed. In a mathematic point of view, a transform is a two-by-three matrix which is multiplied by a vector. (which is the result of the sum of all variation functions within the transform). Apophysis tries to abstract this mathematical aspect by presenting the three columns of the matrix as the three points of a triangle. This way you are able to control the calculation visually. Moving, rotating or scaling a triangle has a similar effect to the transformation matrix.

 

A transform can take several properties:

 

Weight

The transform weight is a value which indicates how "important" the transform (and its variations) is. Higher values on one transform emphasizes the selected one where every other transform is faded. In Apophysis, the weight can range from almost-zero (0.000001) to 1,000. The default weight for each new transform is 0.5.

Additionally, Apophysis allows to define the weights in a more detailed way. This feature was named "Xaos" and you can learn more about it in the Xaos-topic.

 

Color

The transform color defines the color of the points generated by the corresponding transform. This must not be the exact color indicated. The finally selected color is also determined by the color speed parameter. To learn more about coloring your flames, read the coloring topic.

 

Color speed

The color speed is a parameter which supports the color distribution. The higher it is, the more the transform inherits colors from other transforms. In Apophysis, the color speed ranges from -1 (no color variation at all within the transform) to 1. (transform completely inherits other transforms colors) More about the color speed in the coloring topic.

 

Opacity

The transform opacity defines how much visible the points calculated by the corresponding transform are. In Apophysis, this value can range from zero (invisible) to one. (visible) As fractal flames consist of repeated shapes, the points of the corresponding transform may be copied by another transforms. These repeated shapes are not influenced by the opacity of the source transform but by the one of the transform which repeated the shapes.

 

Post-transform

A post-transform is an additional triangle which defines a transformation which is applied after the points are calculated. A the "normal" triangle can change the shape generated, the post-transform triangle does not. So this feature allows you to move, rotate, scale or shear the shape afterwards without changing its general shape.

 

In addition to normal transforms, Apophysis also supports one final transform to be applied. This special transform has influence over all transforms and its variations are applied to the overall shape generated by the other transforms

Variations

 

Variations in Apophysis are essential for the vast amount of different types of fractal flames. There are many variations built into Apophysis and even more available as plugins. They define the nature and the look of the shape which is generated by a transform.

 

A variation is a mathematical function which can a assigned to transforms. Assigning multiple variations to a transform combines them by using the sum of all assigned variations. A useful extension to variations are variables. Variables can provide detailed control about the variations functionality in addition to changing the transformation triangle positions.

 

Some built-in variations have specific prefixes which are important for the order of execution. Variations prefixed with "pre_" are applied before any other calculation is made, those prefixed with "post_" are applied after all calculations are done. Popular examples of this kind of variations are "pre_blur" and the "post_rotate" variations.

 

You can find a catalog of common variations and corresponding example images here. If you want to learn how a certain plugin variation works or looks, check the provided Readme-file or the description provided by the author.

Coloring

 

Contrary to common belief, coloring fractal flame does not work like in graphic applications like GIMP or Photoshop. It is rather the selection or creation of a color map and the definition of two parameters for each transform: color position and color speed.

 

The color position is simply explained: take the amount of colors in the color map (256 in current Apophysis releases) and multiply it with the color position. (between zero and one) The result is the color index where you can find the actual color in the color map. This color is the base color of the transform. It is indicated at the color tab of the editor window and updates when you select a color.

 

The color speed however is difficult to explain and understand. The easiest way is to imagine it as an index for the color variation on a shape generated by the corresponding transform and a value indicating how much other transforms can dictate a color. There are three key values between which the effects are blended:

 

-1

The transform does not inherit any color given by other transforms and shows no variation of colors. The entire shape has exactly the color defined by the color position.

 

0

The transform shape color differs from the color defined by the color position. The difference is dependend from the color speed of other transforms.

 

1

The transform does not have any influence on its shapes color. All colors are inherited from other transforms.

 

Apophysis currently features 700 built-in color maps and offers the import of image files to generate new color maps as well as loading gradients from UltraFractal. You can also adjust loaded gradients in the adjustment window.

Xaos

 

Xaos is a relatively new feature in Apophysis. The artificial word "Xaos" abstracts either the term "controlled chaos" or "weight modifier" which is also describing this feature. In addition to setting a single weight to a transform you can also tell the algorithm how much a transform is visible to other transforms. This is done on the Xaos-tab within the editor window. Apophysis uses a so called "Chaos game" to determine the order in which the transforms are applied into the overall calculations. During this, a random transform is selected and its transformation matrix and variations are applied to the recently plotted point to determine the next point to be plotted. With Xaos, you can take influence on this selection.

 

You can view the Xaos-links through two point of views. The default mode is showing how much visible the current transform is to other transforms while the alternate mode ("View as from") shows the opposite direction - how much visible other transforms are to the current transform.

 

Xaos is mainly used to manipulate the output of transforms. The main use however is advertised through a feature called linked transforms. Linked transforms act as an additional layer which "represents" the source transform after a new set of variations and transformations were applied. A linked transform is much like a post-transform but with the additional possibility to add variations to it.
