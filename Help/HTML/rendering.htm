Rendering window

The render window

 

The render window offers you a vast amount of options to turn your flame into an image file. The "Output" tab contains status messages and render statistics for the currently done or last completed render job. The "Settings" tab contains six regions:

 

Destination

You can enter the path of the image file to be created here. There are three formats which can be written by Apophysis:

 

Windows Bitmap (.bmp)

A lossless format which can provide a maximum of quality but does not support transparency. Use this format if you want to render very large images without transparency.

(e.g. for print)

 

JPEG File Interchange Format (.jpg)

A compressed format which loses quality through a lossy compression algorithm and does not support transparency but is the only format which can contain metadata in the current versions of Apophysis 7X. Use this format if you do not plan to further edit the rendering result with an image processing application such as Photoshop or GIMP and if you don't want to print the result.

 

Portable Network Graphics

A compressed but lossless format which supports transparency but tends to fill up your system memory when rendering at very large resolutions. Use this format if you need transparency.

 

The folder button on the right opens a dialog where you can browse to the destination directory and specify a file name. The arrow button opens a new explorer window at the target directory.

 

Size

This region lets you specify the dimensions of the output image in pixels. Check the box "Keep aspect ratio" to preserve the ratio between width and height when changing the sizes in their boxes.

 

Quality settings

Here you can specify properties related to the output quality. The field "quality" contains the point density of the output image. Setting it too low produces a grain effect on the edges of your flame while setting it too high may increase your render time without earning a better output quality. The point density should be set based on the impression the main preview gives you. If the main preview contains strongly visible grainy regions even when setting the quality in the toolbar drop-down to 1000, you should consider setting a value larger than 4000 to ensure your output image contains a minimum of grainy regions.

The oversample field specifies the amount at which the resolution is multiplied internally before it will be filtered and scaled down to the user-specified dimensions. The strength of the filtering process is specified through the filter radius-field.

 

Resource usage

This region gives you an overview of the amount of system memory used during the rendering process. If the bar is filled up more than 80%, consider activating the memory limitation feature. This increases the render time but ensures that other applications will have enough system memory to remain working.

The buffer depth-drop-down lets you choose the internal size of a calculated point in memory. Higher buffer depths may increase quality but at current releases of Apophysis 7X this may also lead to instabilities. This feature should be used by advanced users only.

 

Output options

In this region you can specify options related to the image output. If you want to store your flame at the same location as the output image, check the box "save parameters".

If you have chosen JPEG as your output format, you may also include metadata such as an author string or the flame data into your image.

 

Completion options

In this region, you can specify whether you want to fine-adjust properties like the background color, the filter radius or the brightness- and gamma-settings after the rendering process is done (through "Post-process after completion"), whether to shut down the computer after rendering or whether you want to save incomplete images at a lower point density when the rendering was cancelled. It is strongly recommended to activate at least the post-process feature to allow you the reviewing of your image as the post-process window can not be opened unless a rendering job completed.
