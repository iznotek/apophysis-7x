Main window

 


The main window of Apophysis 7X.13
 

The main window is the first thing you will see when Apophysis starts. It consists of various areas which are explained below.

 

The menu- and toolbar area at the top
This area offers you full control of all Apophysis features. You can reach every other part of the program from there. Read the topics Main menu and Main toolbar to learn more about this area.

 

The batch listing on the left
All loaded flames in the current batch (collection of flames) are listed in the batch listing. You can use the batch as either a library of your fractal flames or as a collection of different versions of one fractal flame. All entries in this list are saved in a single file.

 

The main preview in the center of the window
This is the largest and most detailed preview panel in Apophysis. You can review your flame before rendering there. Use the quality drop-down in the toolbar to select the point density of the preview (often referenced as "Quality")
When the main preview is updating or a batch file loads, a progress bar will appear on the bottom of the main preview to tell you how far the current operation is done.

 

The status area at the bottom
You can read all important information there. From left to right, there are the following panels:

 

Loading status

If something went wrong during loading a batch you can see it there.

 

Elapsed and remaining times

These two panels show you the elapsed and remaining times of an operation. (like updating the main preview)

 

Selected flame name

This panel shows you the name of the selected flame within the currently loaded batch.

Editor window

 

The editor window

 

The editor window is your central place to design fractal flames. It offers full control over the appearance of your work. It consists of various areas explained below.

 

The toolbar area
In the toolbar, you can access the features of the editor. Alternatively, you can access most of them in the context menu of the editor panel. To learn more about the toolbar features, read the editor toolbar topic.

 

The mini-preview
The mini-preview shows the currently editing flame. It updates way faster than the main preview. Right-clicking lets you choose the quality of the preview.

 

The editor panel
The black area at the window client shows an arrangement of triangles. These represent the transform nodes which control the transformation of the applied variations.

 

The transform selection- and property area
This area consists of a drop-down box where you can select the active transform (alternatively you can click a triangle in the editor panel), a field where you can name your transform and the transform weight box. Below these fields, you can choose from a variety of tabs which group the transform properties into meaningful categories:

 

Variations

Here you can specify the weight of each transforms variation. The list enumerates all built-in variations and those which were loaded through plugins.

Below the list is a button captioned "Clear". This button removes every active variation by setting it to zero.Variations weighted zero are inactive.

Also, you can hide every inactive variation by checking the box captioned "Hide non-used variations". This is useful if you want to see which variations a transform uses quickly without checking each entry in the list. (which can become very long)

To learn more about variations, read the variations topic in the flames section.

 

Variables

Some variations have additional parameters named variables. With variables you can customize the effect of variations. A popular example for a variation with variables is the "julian"-variation.

Below the variables listing is a box captioned "Show all variables". If this box is checked, every variable available within the loaded variations will be listed. By default, only variables of active variations will be shown.

 

Xaos

The Xaos-tab allows you to specify transform weights relative to other transforms. The list contains an entry for each transform in the flame for which you can set the weight of the currently selected transform relative to the corresponding transform in the list.

There are two view modes which you can choose below the list: "View links as to" lets you specify the weight of the current transform to other transforms. "View links as from" lets you specify the weight of other transforms to the current transforms. To learn more about Xaos read the Xaos topic in the flames section.

 

Triangle

This tab offers you a variety of controls to move, rotate and scale the triangle representing the current transform. The numbers on top let you enter exact transform coordinates while the controls below let you alter them visually:
 

•Rotation: The angled arrows rotate the triangle by 90 degrees clockwise or counterclockwise while the curvy arrows rotate it by the angle you specified in the drop-down.
•Movement: Each arrow moves the triangle by the amount you specified in the drop-down in the chosen direction.
•Scaling: The drop-down specifies the percentage for which the current transform should be scaled. The left icon scales it down while the right scales it up.
 

Below these controls is a set of icons:
 

Stores the current transform coordinates in the clipboard

Pastes the transform coordinates from the clipboard into the current transform. This will only work when there is valid data in the clipboard.

Enables or disables the extended edit mode. When the extended edit mode is activated, you can move, rotate and scale the transform with additional grips in the editor panel.

Locks or unlocks the transform axes when rotating a transform. (lines from O to X and Y)

Enables or disables the automatic balancing of transform weights.

Enables or disables post-transform editing. (to learn more about post-transforms read the transforms topic in the flames section)

 

The controls on the very bottom let you take influence on the so called pivot point. The pivot point is the location around which transforms are rotated and which is the center of a scaling operation. You can either enter the coordinates of this point into the two boxes (left: X, right: Y) or use the buttons below. The button captioned "R" resets the pivot point to its default location, the large button right to it toggles between local pivot (point O of the triangle) and the world pivot. (point at the coordinates 0,0) The button captioned "P" lets you pick the pivot point using the mouse.

 

Transform

The transform tab contains input boxes similar to those on the triangle tab at its top. The difference is that the values for are vectors rather than absolute point coordinates. This means that the values you enter for X and Y are relative to those of O. The buttons containing the caption of the vectors will reset the corresponding values to their default once you click them. The large button captioned "reset transform" will reset the entire triangle to its default coordinates.

With the buttons captioned "rectangular" and "polar" you can choose the coordinate system type for the edit operation. The polar coordinate system uses degrees as unit.

The set of boxes and buttons below is similar but for the post-transform triangle.

At the very bottom is a box captioned "Auto-calculate pre_zscale" which automatically sets the built-in variation "pre_zscale" based on the transform vectors. This is useful if you plan to create a 3D flame.

 

Colors

The color tab offers you controls to colorize your flame. You can specify the base color with the slider on the top. The color speed (formerly called symmetry) specifies the amount of which the color of the current transform is dependend from other transforms. This ranges from -1 (completely independend) to 1. (transform fully inherits colors from other transforms) The default value is zero.

The opacity field specifies how much visible the variations of the current transform are on the image. If you set zero, the transform itself is invisible but its influence on other transforms is still visible as well as eventually repeated versions of the transform.

The box captioned "solo" hides all other transforms (opacity is set to zero) and only shows the current one. This effect is not permanent and can be undone any time - even when you close and restart Apophysis.

The sliders below offer display controls about the variation preview feature you can activate in the toolbar. They have no effect on the rendered image.

To learn more about coloring your flames read the coloring topic in the flames section.

Adjustment window

 

The adjustment window

 

The adjustment window offers controls to change overall properties within the selected flame. From here you can change the camera settings, set a global color map and adjust the flame brightness and gamma values.

 

In the top section you will find a mini preview (clicking and dragging changes the 3D-properties of the camera - unlike the main preview which changes the 2D-properties) and a set of boxes where you can input the camera properties:

 

Depth blur

This field controls the optional depth-of-field effect you can set up for your 3D-flames. This only works properly if you adjusted pitch and yaw first. Setting it to zero disables the effect.

 

Pitch and Yaw

These two fields allow you to rotate the camera in the 3D space. If you set both to zero, the camera will look from top down to your flame.

 

Height

This field lets you adjust the vertical offset of the base plane of your flame. Positive values move it up, negative values move it down. By default, this offset is zero.

 

Perspective

With this field you can set the perspective distortion of your flame. It is commonly used to strengthen the 3D-effect of flames which where adjusted using pitch and yaw.

 

Scale

This field sets the point scale relative to the image size. It acts as a zoom feature but higher values will decrease the overall quality of the rendered image. If you want to preserve the quality, use the zoom feature instead. If you want to preserve the render speed, use this field.

 

 

The bottom section features a set of tabs which group several features into meaningful categories:

 

Camera

This tab contains the 2D-related camera controls.

The "zoom"-slider offers a zoom feature which preserves the flame quality but drastically decreases the render speed when using values greater than zero.

The X- and Y-position-sliders let you pan the camera after the 3D-transformations have been applied while the rotation-slider lets you rotate the camera after the values given in pitch and yaw have been applied.

 

Rendering

This tab contains fields which let you control the properties of image generation.

The gamma-slider lets you adjust the gamma-correction in order to improve the visibility of fine structures while the brightness-slider lets you adjust the overall brightness of the image. The vibrancy-slider lets you control the richness of the flames colors. You can also set the flame background color here.

The gamma treshold-box lets you fine-adjust the gamma correction in order to reduce the grain effect occuring on lower density areas. Higher values tend to make the image darker.

 

Gradient

The gradient tab offers you a powerful tool to create or adjust a gradient used as the main color map for your flame. On this tab, there is a black drop-down which provides a vast amount of built-in gradients. The slider above can take various modes which let you adjust the selected gradient. Click the text on the left of this slider to select the edit mode. With the text field on the right you can enter a value for the slider using your keyboard rather than adjusting it with your mouse using the slider. The button captioned "reset" restores the state of the gradient prior to opening the gradient tab or selecting an edit mode.

The top section of this tab contains a preview of the current gradient. Clicking and dragging on this section rotates the gradient even if you selected a different edit mode in the field below.

On the bottom right you will find a set of buttons:

 

Copies the current gradient into the clipboard (UltraFractal-formatted)

Pastes the current gradient from the clipboard. This does only work if there is a valid UltraFractal-formatted gradient in the clipboard. You may also copy a gradient from UltraFractal and paste it into Apophysis.

Opens the gradient browser in which you can open, save and explore a gradient file.

Lets you open an image and create a gradient from its colors.

 

Image size

This tab is grouped into three columns. The one on the left lets you enter an  size for the main preview, the one in the middle lets you manage up to three presets for the main preview sizes (click the wide button to load a preset or the button right next to it to save it) and the one on the right lets you apply your selected size. If the box on the right (captioned "resize main window") is not checked, the size image will be centered within the main preview area, elsewise the main window will be resized in order to match the preview size you entered.

Rendering window

 

The render window

 

The render window offers you a vast amount of options to turn your flame into an image file. The "Output" tab contains status messages and render statistics for the currently done or last completed render job. The "Settings" tab contains six regions:

 

Destination

You can enter the path of the image file to be created here. There are three formats which can be written by Apophysis:

 

Windows Bitmap (.bmp)

A lossless format which can provide a maximum of quality but does not support transparency. Use this format if you want to render very large images without transparency.

(e.g. for print)

 

JPEG File Interchange Format (.jpg)

A compressed format which loses quality through a lossy compression algorithm and does not support transparency but is the only format which can contain metadata in the current versions of Apophysis 7X. Use this format if you do not plan to further edit the rendering result with an image processing application such as Photoshop or GIMP and if you don't want to print the result.

 

Portable Network Graphics

A compressed but lossless format which supports transparency but tends to fill up your system memory when rendering at very large resolutions. Use this format if you need transparency.

 

The folder button on the right opens a dialog where you can browse to the destination directory and specify a file name. The arrow button opens a new explorer window at the target directory.

 

Size

This region lets you specify the dimensions of the output image in pixels. Check the box "Keep aspect ratio" to preserve the ratio between width and height when changing the sizes in their boxes.

 

Quality settings

Here you can specify properties related to the output quality. The field "quality" contains the point density of the output image. Setting it too low produces a grain effect on the edges of your flame while setting it too high may increase your render time without earning a better output quality. The point density should be set based on the impression the main preview gives you. If the main preview contains strongly visible grainy regions even when setting the quality in the toolbar drop-down to 1000, you should consider setting a value larger than 4000 to ensure your output image contains a minimum of grainy regions.

The oversample field specifies the amount at which the resolution is multiplied internally before it will be filtered and scaled down to the user-specified dimensions. The strength of the filtering process is specified through the filter radius-field.

 

Resource usage

This region gives you an overview of the amount of system memory used during the rendering process. If the bar is filled up more than 80%, consider activating the memory limitation feature. This increases the render time but ensures that other applications will have enough system memory to remain working.

The buffer depth-drop-down lets you choose the internal size of a calculated point in memory. Higher buffer depths may increase quality but at current releases of Apophysis 7X this may also lead to instabilities. This feature should be used by advanced users only.

 

Output options

In this region you can specify options related to the image output. If you want to store your flame at the same location as the output image, check the box "save parameters".

If you have chosen JPEG as your output format, you may also include metadata such as an author string or the flame data into your image.

 

Completion options

In this region, you can specify whether you want to fine-adjust properties like the background color, the filter radius or the brightness- and gamma-settings after the rendering process is done (through "Post-process after completion"), whether to shut down the computer after rendering or whether you want to save incomplete images at a lower point density when the rendering was cancelled. It is strongly recommended to activate at least the post-process feature to allow you the reviewing of your image as the post-process window can not be opened unless a rendering job completed.

Mutation window

 

The mutation window

 

The mutation window offers an easy way to change the appearance of your flame. This feature is a remainder of older versions of Apophysis and is not used commonly anymore as you have relatively less influence on how your flame looks later compared to the editor window.

 

The client of this window consists of nine fields where the one in the middle contains a preview of your current flame. The fields around the middle offer you possibilities of mutation directions. Click one of them to apply the shown mutation to your current flame. Click the center to generate a new set of mutation directions.

 

With the slider captioned "speed" on the bottom you can specify the amount of which the flame changes after each mutation while the drop down below offers you a list of variations you can base the mutation on. This list contains all currently loaded variations plus an entry captioned "random" which makes the mutation select a random variation to apply for each direction in each set.

If you check "same no. of transforms" the variations will be applied on the existing set of transforms, elsewise a new transform will be added for the mutation.

UPR options

 

 

This tab offers options to export Apophysis flames into UltraFractal parameters. It is for advanced users and you should consult the UltraFractal-manual to learn about the effects of individual options.

 

See also: General - Editor - Display - Random - Variations - Gradient - UPR - Paths - Autosave

 

Search paths

 

 

Default parameter file

Specifies the batch file which should be loaded after Apophysis started. If you leave this field empty, a random batch is generated after each start.

Check the field "Remember last opened file" to re-open the batch file you have loaded before you closed Apophysis.

 

Smooth palette file

The file where gradients imported from images are saved into.

 

Function library

A file which is internally included into every script.

 

Export renderer

The path to the renderer which should be used by the "Export flame"-function in the file menu of the main window.

 

Help file

The help file which should be opened by the "Content" entry in the help menu of the main window. By default, the path to this file is set.

 

See also: General - Editor - Display - Random - Variations - Gradient - UPR - Paths - Autosave

 

Autosave options

 

 

File name

The path to the batch file where the automatically saved flames should be stored in.

 

Save frequency

The interval in minutes at which Apophysis should store the currently selected flame into the batch file specified by the field above.
