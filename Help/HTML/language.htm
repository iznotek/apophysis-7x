Identifiers

 

Identifier names in scripts (variable names, function and procedure names, ...) should begin with a letter (a..z or A..Z) or an underscore character ("_") and can be followed by digits, letters or the underscore character. They can't contain any other character or spaces.
 
Valid identifiers:
 
VarName
_Some
V1A2
_____Some____
 
Invalid identifiers:
 
2Var
My Name
Some-more
This,is,not,valid

Literals and Operators

 

There is a number of operators in the script language:

 

•Basic operations (addition, subtraction, multiplication, division)
 
a := b + c;
c := a - b;
d := a * 2;
a := d / 2;
 
Note that the + operator also can be used to chain strings.
 
•Assignation and comparison
 
a := 'This text is going to be assigned to a';
if (a = b) then ShowMessage('This message pops up when a is equal to b');
if (a <> c) then ShowMessage('This message pops up when a is not equal to c');
 
Do NOT swap := and = !
Other valid comparison operations are < and > - but only for numbers.
 
•Modulo and integral division operation
 
a := 5; b := 2;
c := a mod b; // c will be 1
ShowMessage('The remainder of 5 / 2 is ' + IntToStr( c ));
 
d := a div b; // d will be 2
ShowMessage('The result of the integral division of 5 and 2 is ' + IntToStr( d ));
 
•Indexing operation
 
a := [ 1, 2, 3 ];
ShowMessage('The second element of a is a[1] (zero-based counting): ' + IntToStr(a[1]));
 
•Boolean operations
 
a := ((c = 1) and (b <> 0)) or (not some_boolean_variable)

 

Valid literals are numbers, strings and array constructors:

 

a := 'I am a string literal';

b := 2;

c := [ 1, 2, 3 ];

 

Literals are constant and may not be located at the left side of an assignation operation.

 

Arrays may contain multiple dimensions:

 

a := [ [1, 2, 3], [4, 5, 6] ];

ShowMessage(IntToStr( a[0, 1] );

Conditions and Loops

 

Apophysis script supports conditional execution of code through branching statements:

 

if J <> 0 then Result := I/J;
if J = 0 then Exit else Result := I/J;
if J <> 0 then
begin
Result := I/J;
Count := Count + 1;
end else begin
Done := True;
end;

 

case fruit of
'lime': ShowMessage('green');
'orange': ShowMessage('orange');
'apple': ShowMessage('red');
else
ShowMessage('black');
end;
 

Please note that a semicolon (";") is not allowed directly before the "else"-keyword.

 

Additionally, you can define loops through the following statements:

 

while Data[I] <> X do I := I + 1; 
while I > 0 do 
begin 
  I := I div 2; 
  X := Sqr(X); 
end; 

 

repeat
K := I mod J;
I := J;
J := K;
until J = 0;
 
continue_loop := true;
repeat
continue_loop := InputQuery('My Script', 'Enter a value (0..9): ', I);
until ((I >= 0) and (I <= 9)) or (not continue_loop);
 
for i:=a to b do
begin
j:=i^2;
sum:=sum+j;
end;

Procedures and Functions

 

Apophysis script supports the definition of own sub-programs with either no return value (procedures) or with a return value. (functions) You may define those as following:

 

procedure DoSomething;

begin

  // your code here

end;

 

function GetSomething;

begin

  // the variable "result" is auto-defined in functions

  Result := 0;

end;

Comments

 

Comments can assist you and others while reading the script. There are three types of comments:

 

•Single-line comments (starting with "//")
•Block comments (starting with either "(*" or "{" and ending with "*)" or "}")
 
//This is a comment before ShowMessage 
ShowMessage('Ok'); 
 
(* This is another comment *) 
ShowMessage('More ok!'); 
 
{ And this is a comment 
with two lines } 
ShowMessage('End of okays');
